สถาปัตยกรรม 3-Tier Architecture

ในหลักสูตรนี้ เราจะเรียนรู้การพัฒนาซอฟต์แวร์แบบ 3-Tier Architecture ซึ่งแยกการทำงานออกเป็น 3 ชั้นหลัก

Tier 1: Presentation

Frontend / UI Layer

ส่วนที่ผู้ใช้เห็นและโต้ตอบได้

React.js, HTML, CSS, JavaScript

Tier 2: Application

Business Logic Layer

ประมวลผลและจัดการตรรกะทางธุรกิจ

Node.js, Express.js, APIs

Tier 3: Data

Data Storage Layer

จัดเก็บและจัดการข้อมูล

MongoDB, MSSQL

ประโยชน์ของ 3-Tier Architecture

คำถามที่ 1: SDLC ย่อมาจากอะไร?

A) Software Development Language Course
B) Software Development Life Cycle
C) System Development Life Cycle
D) Software Design Learning Center
ถูกต้อง! SDLC หมายถึง Software Development Life Cycle ซึ่งเป็นกระบวนการพัฒนาซอฟต์แวร์ตั้งแต่การวางแผน การวิเคราะห์ การออกแบบ การพัฒนา การทดสอบ และการบำรุงรักษา

คำถามที่ 2: ใน 3-Tier Architecture, Tier ไหนที่รับผิดชอบเรื่อง User Interface?

A) Presentation Tier (Tier 1)
B) Application Tier (Tier 2)
C) Data Tier (Tier 3)
D) ทุก Tier รับผิดชอบร่วมกัน
ถูกต้อง! Presentation Tier (Tier 1) เป็นชั้นที่รับผิดชอบเรื่อง User Interface ซึ่งเป็นส่วนที่ผู้ใช้มองเห็นและสามารถโต้ตอบได้ เช่น HTML, CSS, JavaScript, React.js

คำถามที่ 3: Node.js คืออะไร?

A) เป็น Web Browser ใหม่
B) เป็น JavaScript Runtime Environment
C) เป็น Database Management System
D) เป็น Code Editor
ถูกต้อง! Node.js เป็น JavaScript Runtime Environment ที่ทำให้เราสามารถรัน JavaScript นอก Web Browser ได้ ทำให้สามารถใช้ JavaScript พัฒนา Server-side Application ได้

คำถามที่ 4: Git ใช้สำหรับอะไร?

A) Version Control System
B) Code Editor
C) Web Server
D) Database
ถูกต้อง! Git เป็น Version Control System ที่ช่วยในการติดตามการเปลี่ยนแปลงของ Source Code, การทำงานร่วมกันในทีม, และการจัดการประวัติการพัฒนา

คำถามที่ 5: ข้อใดต่อไปนี้เป็นประโยชน์ของ 3-Tier Architecture?

A) ทำให้โปรแกรมทำงานเร็วขึ้นเท่านั้น
B) ลดการใช้พื้นที่ Hard Disk
C) แยกหน้าที่การทำงานชัดเจน ขยายระบบและดูแลรักษาง่ายขึ้น
D) ใช้ภาษาโปรแกรมได้เพียงภาษาเดียว
ถูกต้อง! 3-Tier Architecture ช่วยแยกหน้าที่การทำงานอย่างชัดเจน (Separation of Concerns), ทำให้ขยายระบบได้ง่าย (Scalability), ดูแลรักษาและแก้ไขง่ายขึ้น (Maintainability), และทีมสามารถทำงานแยกกันได้

แบบฝึกหัดเพิ่มเติม

แบบฝึกหัดที่ 1: การตั้งค่า Git

ให้นักศึกษาทำตามขั้นตอนต่อไปนี้:

  1. สร้างโฟลเดอร์ใหม่ชื่อ "my-first-project"
  2. เปิด Terminal/Command Prompt ในโฟลเดอร์นั้น
  3. รัน git init เพื่อเริ่มต้น Git repository
  4. สร้างไฟล์ index.html ด้วย Hello World
  5. ใช้ git add . เพื่อเพิ่มไฟล์เข้า staging area
  6. ใช้ git commit -m "Initial commit" เพื่อ commit

แบบฝึกหัดที่ 2: การใช้งาน VS Code

ให้นักศึกษาลองใช้ Extensions ที่สำคัญ:

  • Live Server: สำหรับรัน HTML แบบ real-time
  • GitLens: สำหรับดู Git history
  • Prettier: สำหรับจัดรูปแบบ code
  • Auto Rename Tag: สำหรับ HTML tags

แบบฝึกหัดที่ 3: สร้าง Package.json

  1. สร้างโฟลเดอร์ใหม่สำหรับโปรเจกต์ Node.js
  2. รัน npm init และกรอกข้อมูล
  3. ติดตั้ง package เช่น npm install express
  4. สร้างไฟล์ server.js ด้วยตัวอย่าง Express server
  5. เพิ่ม script ใน package.json: "start": "node server.js"
  6. รัน npm start เพื่อทดสอบ
การดำเนินการสำเร็จ!
